About this Site's Design and Development

This site is a collaborative effort between designer Boelts Design, Inc. and developer Lisantra Technologies, LLC for James A Cook, all in Tucson, Arizona.

Design

Development

Lead Developer Michael Gatto comments on his choice of technique:

"I really enjoyed working on this site; I became familiar with James' amazing interpretations of culture through sculpture. Before, I hadn't been able to 'get it', but adding James' writeup to each sculpture page was a treat. From a technical perspective, I also had the opportunity to depart from my normal routine of heavy back-end programming and make some beautiful markup."

"From the very first content review, I knew this site would have many repititious pages. I had to choose between a dynamically scripted site and a static Html site. I basically had already decided I wanted to depart from my usual Php programming and concentrate on advanced markup. However, developer efficiency is always paramount, so I immediately turned to Dreamweaver's templating system."

"On this site, I decided to use the revived @font-face method of including arbituary fonts in a webpage. This way, I was able to faithfully reproduce Jackson Boelts' design inent. Finding an open source font which matched his design proofs was easy with Font Squirrel."

"Although I've been working in Dreamweaver for about 7 years, I pushed Dreamweaver CS4's templating language beyond its limits. To address some of its shortcomings, I even investigated some PERL and Python packages. One major shortcoming was nested templates not working with imported Xml content."

"I managed the source code in Subversion, which also allowed me to depart from my typical usage of Mercurial as a Version Control System. I also experimented with advanced Subversion branching and merging techniques, even running into a long-standing bug involving mergeinfo metadata in a repository."

"I also had to do a considerable amount of media conversion such as re-encoding video from gigantic Quicktime .mov files to much smaller Flash Video .f4v files. I tried some newer formats such as H264 and Google's brand-new WebM. H264 files ended up being almost as large as the Quicktime sources, while WebM is only supported in the Google Chrome browser, so we stuck to the tried and true Flash Video. I used FlowPlayer to play the videos online."

"I tested the site on a virtual machine to Although I was tempted at points to just use Php, I stuck with the plan and am quite satisfied at how it turned out. I hope you are, too."

Technologies

This site uses open-source internet technologies, web standards and clear design. Among them are:

  • XHTML 1.0 (hand coded)
  • CSS 2.1 (hand coded, with a sprinling of CSS3)
  • Blueprint 1.0 (CSS Framework)
  • Web Fonts (hand coded)
  • jQuery 1.4.2 (JavaScript Framework)
  • Tested on Apache 2.2
  • Using Ubuntu Server 9.10
